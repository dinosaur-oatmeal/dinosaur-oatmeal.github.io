American Sign Language (ASL) Recognition

A real-time computer vision and machine learning pipeline built to explore data collection and live inference in real-world conditions.

What I Learned

Dataset construction under constraints: built a custom dataset of 2,000+ labeled ASL samples, highlighting real data collection difficulties.
Evaluation beyond static testing: validated a TensorFlow model (90%+ accuracy) under varying lighting, pose, and motion conditions, focusing on the gap between test-set metrics and live inference.
Noise-aware inference design: implemented motion-stabilization gating for stable gesture recognition, reducing spurious classifications.
Human-centric ML: designed real-time confidence threshold controls and asynchronous text-to-speech output, focusing on real-world use-cases.

Neural Network from Scratch (Foundational Modeling)

A ground-up implementation of a feedforward neural network designed to make model behavior explicit, inspectable, and explainable.

What I Learned

Mechanistic understanding of learning: implemented forward propagation, backpropagation, and gradient descent directly in NumPy, making parameter updates traceable.
Evaluation without abstractions: trained on the MNIST dataset (95%+ accuracy) using custom architecture design, weight initialization, and manual hyperparameter tuning.
Model analysis: visualized loss curves, weight evolution, and accuracy trajectories across epochs using Matplotlib to observe learning dynamics.
Explanation-first design: structured the project as a code-first, math-supported walkthrough, with an emphasis on modeling interpretability.

Algorithm Learning Platform (Pedagogical Visualization)

An interactive educational platform designed to make algorithmic processes inspectable and understandable.

What I Learned

Interpretability as a design problem: visualized 20+ algorithms and data structures step-by-step, emphasizing the inner-workings of common functions.
Communication-first controls: implemented buttons such as pause, reset, and play to support different learning styles and paces.
Functional paradigm programming: built entirely in Elm using core functional programming principles, enforcing immutability and explicit state transitions.
Proven instructional use: adopted by UTA faculty and used by 50+ students per semester in live lectures and self-paced studying.

Queen's Logic Puzzle Generator & Game

A full-stack puzzle system focused on constraint satisfaction problems and unique board generation.

What I Learned

Constraint satisfaction & verification: designed a bespoke flood-fill–based board generator that guarantees exactly one valid solution per puzzle.
System modularity: implemented a FastAPI + Pydantic backend for puzzle generation and validation, decoupled from the SVG-based client.
Production deployment: deployed as a Docker container with rate limiting, CI/CD, and automated redeployments on GitHub push.
